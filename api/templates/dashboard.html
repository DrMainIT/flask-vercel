Come funziona un sito web

Un sito web è diviso in tre parti principali:

  • Frontend: Gli elementi visivi (pulsanti, testo, grafiche) con cui gli utenti interagiscono.
  • Backend: La logica che elabora i dati dal frontend e fornisce risposte.
  • Database: Una raccolta di dati organizzati per memorizzare e recuperare informazioni.

Frontend

Il frontend include tutti i componenti visivi che gli utenti vedono e con cui interagiscono, come ad esempio:

  • Pulsanti



  • Form:



  • Testo e immagini

Backend

Il backend gestisce la logica del sito. Elabora i dati dal frontend ed esegue azioni come:

  • login di un sito
  • Interagire con il database per recuperare o salvare informazioni
  • elaborare i dati

Alcuni framework comuni per il backend includono:

  • Flask (usato in questo progetto)
  • Django
  • FastAPI
  • Express.js
  • Spring

Database

Un database viene utilizzato per memorizzare e recuperare dati, come i dettagli dell'utente o l'inventario dei prodotti. In questo progetto, usiamo SQLite, che è leggero e facile da configurare.

Collegare tutto

Il backend collega il frontend e il database. Ad esempio:

Un utente compila un form per registrarsi. Il backend elabora i dati e li salva nel database.

Richieste HTTP

La comunicazione tra il frontend e il backend avviene tramite richieste HTTP. Le più comuni sono:

  • GET: Recupera dati
  • POST: Invia dati
  • PUT: Aggiorna dati
  • DELETE: Rimuove dati

Passare online

Affinché un sito web sia accessibile online, necessita di un dominio (un nome univoco) e un indirizzo IP. Ad esempio, https://google.com è un dominio che punta ad un server di Google tramite un indirizzo IP. ovvero 172.217.20.206se si prova a digitare l'ip sulla barra di ricerca comparirà il sito di google

Configurazione

Per creare questo progetto, abbiamo utilizzato i seguenti strumenti:

  • Python: Installa da python.org.
  • Flask: Installato tramite pip install flask.
  • SQLite: Incluso in Python, non è necessaria un'installazione separata.

2. Route e la loro funzionalità

Route della homepage (/)

La route della homepage renderizza il file index.html:

@app.get('/')
def index():
    return render_template('index.html')

Questo è il punto di ingresso dell'applicazione, che serve la pagina principale di atterraggio.

route di registrazione (/register)

Questa route ha due metodi: GET e POST.

GET: Renderizza il modulo di registrazione register.html

@app.get('/register')
def register():
    return render_template('register.html')

POST: Gestisce l'invio del modulo, sanifica i dati dell'utente e li inserisce nel database.

@app.post('/register')
def register_post():
    data = request.form
    name = data['username']
    email = data['email']
    password = data['password']
    conn = get_db_connection()
    conn.execute('INSERT INTO users (name, email, password) VALUES (?,?,?)', (name, email, password))
    conn.commit()
    conn.close()
    return render_template('login.html')

Una volta completata la registrazione, l'utente viene reindirizzato alla pagina di login.

route di login (/login)

Questa route ha anch'essa due metodi: GET e POST.

GET: Renderizza il modulo di login login.html

@app.get('/login')
def login():
    return render_template('login.html')

POST: Valida le credenziali dell'utente interrogando il database.

@app.post('/login')
def login_post():
    data = request.form
    email = data['email']
    password = data['password']
    conn = get_db_connection()
    cursor = conn.execute('SELECT * FROM users WHERE email = ? AND password = ?', (email, password))
    users = cursor.fetchall()
    conn.close()
    if len(users) > 0:
        return render_template('dashboard.html')
    else:
        return render_template('login.html')

Se le credenziali corrispondono a un utente esistente nel database, l'utente viene indirizzato alla pagina dashboard.html. Altrimenti, viene reindirizzato al modulo di login.

Connessione al database

Una funzione di supporto get_db_connection() viene utilizzata per stabilire e restituire una connessione al database SQLite:

def get_db_connection():
    conn = sqlite3.connect('mydatabase.db')
    print("Database aperto con successo")
    return conn

Questa funzione garantisce che tutte le interazioni con il database siano centralizzate per coerenza e per un debug più facile.
